# ADR-001: Модульный монолит

## Status

Accepted

## Context

Домашняя self-hosted WMS должна быть простой в разработке, деплое и сопровождении одним человеком. Enterprise-паттерны (микросервисы, брокеры) избыточны.

## Decision

Использовать один Go-сервис (модульный монолит) с REST API и PostgreSQL. Модули изолированы на уровне пакетов, но развёртываются как единое приложение.

## Consequences

**Плюсы:** простой деплой, транзакционная целостность в одной БД, низкий ops-overhead.

**Минусы:** ограниченное независимое масштабирование модулей.

## Rejected Alternatives

- Микросервисы
- CQRS / Event Sourcing
- Kafka-ориентированная архитектура
