# ADR-004: Идемпотентный sync-протокол

## Status

Accepted

## Context

При нестабильной сети неизбежны ретраи и дубликаты запросов. Повторная отправка не должна создавать второе движение остатков.

## Decision

- Каждая мутирующая операция имеет `device_id` + `operation_key` (стабильный ключ ретраев).
- Сервер хранит `payload_hash` и результат применения.
- Повтор с тем же ключом и hash → возврат сохранённого результата (200).
- Повтор с тем же ключом, но другим hash → `409 IDEMPOTENCY_KEY_PAYLOAD_MISMATCH`.
- Область уникальности: `(device_id, operation_key)`.
- Retention: 30 дней (настраивается через config).

## Consequences

**Плюсы:** нет двойных списаний, предсказуемый retry.

**Минусы:** хранение журнала идемпотентности, политика очистки.

## Rejected Alternatives

- Best-effort без ключей идемпотентности
- Дедупликация по времени или эвристикам
