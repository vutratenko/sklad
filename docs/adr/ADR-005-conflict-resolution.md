# ADR-005: Стратегия конфликтов синхронизации (MVP)

## Status

Accepted

## Context

Офлайн-операции с разных устройств могут конфликтовать при слиянии (например, одновременное списание одного остатка).

## Decision

MVP-политика: **server truth + явная ошибка клиенту**.

1. Сервер — источник истины для остатков и движений.
2. При доменном конфликте (недостаток остатка, нарушение инварианта) → `409` с кодом причины.
3. Клиент делает pull свежего состояния и показывает пользователю diff.
4. Пользователь вручную подтверждает повтор или принимает серверное состояние.
5. Для некритичных полей справочников (описание SKU) допустим last-write-wins по `updated_at`.

## Consequences

**Плюсы:** прозрачность, нет «тихой порчи» остатков.

**Минусы:** часть кейсов требует ручного вмешательства.

## Rejected Alternatives

- Слепой Last-Write-Wins для всех сущностей
- CRDT/OT conflict engine
- Автоматическое «умное» слияние без доменных правил
