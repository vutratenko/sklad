# ADR-006: Модель консистентности movements ↔ stock_balances

## Status

Accepted

## Context

Нужны и полная история движений, и быстрое чтение текущих остатков. Два хранилища создают риск расхождения.

## Decision

- **`stock_movements`** — immutable журнал (источник аудита и восстановления).
- **`stock_balances`** — материализованный агрегат для быстрого чтения.
- Обновление `stock_balances` происходит **в той же транзакции**, что и вставка в `stock_movements`.
- При расхождении (диагностика) `stock_balances` можно пересобрать replay-ом журнала.
- Sync-модуль делегирует применение операций модулю `movements`, не дублируя бизнес-правила.

## Consequences

**Плюсы:** быстрое чтение + полный аудит, транзакционная целостность.

**Минусы:** два хранилища требуют дисциплины в одном unit of work.

## Rejected Alternatives

- Только журнал без агрегата (медленные read-запросы)
- Только агрегат без журнала (нет аудита)
- Eventual consistency между journal и balance внутри одного сервиса
